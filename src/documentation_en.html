1. Introduction

The Saudi Regions Widget Enhanced is a comprehensive and optimized JavaScript library designed to provide an easy-to-use solution for integrating Saudi Arabian regions, cities, and districts data into any web application. This widget offers a robust and flexible way to handle geographical data specific to Saudi Arabia, making it ideal for developers building applications that require location-based functionalities, such as e-commerce platforms, service directories, or governmental applications.

With support for CDN delivery, advanced search capabilities, and a responsive design, this widget aims to streamline the development process and enhance user experience. This documentation will guide you through the installation, configuration, and usage of the Saudi Regions Widget Enhanced, providing detailed examples and best practices for integrating it into your projects.

3. Installation

Integrating the Saudi Regions Widget Enhanced into your project is straightforward. Choose from several installation methods depending on your project's needs and development workflow.

Via CDN (Recommended)

The quickest and most recommended way to include the widget is by using a Content Delivery Network (CDN). Simply add the following tags to your HTML:

<!-- CSS -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/YounisDany/saudi-regions-widget@main/dist/saudi-regions-widget.min.css">

<!-- JavaScript -->
<script src="https://cdn.jsdelivr.net/gh/YounisDany/saudi-regions-widget@main/dist/saudi-regions-widget.min.js"></script>

<!-- Or use the bundled file (JS + CSS) -->
<script src="https://cdn.jsdelivr.net/gh/YounisDany/saudi-regions-widget@main/dist/saudi-regions-widget.bundle.min.js"></script>

Via NPM

For projects using Node.js and a package manager like npm, install the widget as a dependency:

npm install saudi-regions-widget

Direct Download

Download the widget files directly from the releases page on GitHub and include them in your project as local assets.

4. Usage

Once installed, the Saudi Regions Widget Enhanced can be easily integrated into your web page. There are two primary ways to use the widget: quick setup for basic needs and detailed setup for advanced customization.

Quick Setup

For rapid deployment, the createQuick method allows you to initialize the widget with minimal configuration:

<div id="saudi-widget"></div>

<script>
// Create the widget with one line of code
SaudiRegionsWidget.createQuick("saudi-widget", {
    language: "en", // 'ar' or 'en'
    rtl: false // Set to true for right-to-left display
});
</script>

Detailed Setup

For more granular control over the widget's behavior, data levels, and event handling:

// Create a new instance
const widget = new SaudiRegionsWidget({
    dataLevel: "complete", // 'regions', 'regions-cities', 'complete'
    language: "en", // 'ar' or 'en'
    rtl: false,
    dataUrl: "https://cdn.jsdelivr.net/gh/YounisDany/saudi-regions-widget@main/dist/data/"
});

// Wait for data to load
widget.on("ready", function() {
    // Create cascading selects (region -> city -> district)
    widget.createCascadingSelects("widget-container");
});

// Listen for events
widget.on("regionChanged", function(data) {
    console.log("Region selected:", data.region);
});

widget.on("cityChanged", function(data) {
    console.log("City selected:", data.city);
});

widget.on("districtChanged", function(data) {
    console.log("District selected:", data.district);
});

5. Configuration Options

The SaudiRegionsWidget constructor accepts an options object that allows you to customize various aspects of the widget's behavior and appearance:

Option Type Default Description
dataLevel String 'complete' Level of geographical data to load
language String 'en' Display language ('ar' or 'en')
rtl Boolean false Right-to-left text direction
searchEnabled Boolean true Enable search functionality

9. Data Levels

The widget offers different data levels to optimize performance based on your application's needs:

Data Level Size Description
regions ~2KB Loads only the 13 Saudi regions
regions-cities ~85KB Loads regions and their respective cities
complete ~1.2MB Loads regions, cities, and all districts

11. Application Examples

The Saudi Regions Widget Enhanced is framework-agnostic and can be integrated into various JavaScript frameworks:

React Integration

import { useEffect, useRef } from 'react';

function SaudiRegionsComponent() {
    const widgetRef = useRef(null);
    
    useEffect(() => {
        const widget = window.SaudiRegionsWidget.createQuick('saudi-widget', {
            language: 'en',
            rtl: false
        });
        
        widget.on('regionChanged', (data) => {
            console.log('Region changed:', data);
        });
        
        return () => widget.destroy('saudi-widget');
    }, []);
    
    return <div id="saudi-widget" ref={widgetRef}></div>;
}

Vue.js Integration

<template>
    <div id="saudi-widget"></div>
</template>

<script>
export default {
    mounted() {
        this.widget = window.SaudiRegionsWidget.createQuick('saudi-widget', {
            language: 'en',
            rtl: false
        });
        
        this.widget.on('regionChanged', this.handleRegionChange);
    },
    
    beforeDestroy() {
        if (this.widget) {
            this.widget.destroy('saudi-widget');
        }
    },
    
    methods: {
        handleRegionChange(data) {
            console.log('Region changed:', data);
        }
    }
}
</script>
